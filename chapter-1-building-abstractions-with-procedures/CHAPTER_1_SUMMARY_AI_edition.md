# 第 1 章总结 — 用过程构造抽象

> AI 辅助整理版。基于 1.1、1.2、1.3 笔记的总收束。

---

## 一句话概括

> **第 1 章只讲一件事:抽象 (abstraction)。**
> **三节分别在三个不同层级反复演练同一动作 — 识别模式 → 给它命名 → 建立可组合的系统。**

```
1.1  抽象"值"        — 给运算命名         (cube, square)
1.2  抽象"行为"      — 给过程的形状命名    (linear, iterative, tree-recursive)
1.3  抽象"抽象本身"  — 给"过程的模式"命名   (sum, fixed-point, average-damp)
```

**每一节都把"被命名的东西"抽象一级**,直到 1.3.4 的 `fixed-point-of-transform` — 一个**返回函数的函数被参数化的元方法**。

---

## 第一章的核心反射

学完第 1 章后,你脑里应该有一个反射,每写一段代码就触发:

> **"我刚才复制了一段代码,只改了几个名字 — 这几个被改的东西能不能变成参数?"**

- 改的是**数** → 普通参数 (1.1.4 教的)
- 改的是**动作** → **高阶参数**,过程作参数 (1.3.1 教的)
- 想做"造一个能造它的工厂"? → 过程作返回值 (1.3.4 教的)
- 两段抽象本身长得一样? → **再抽象一层** (1.3.4 末尾)

这个反射一旦内化,**代码会越来越短,越来越通用**。

## 三节的核心收获

### 1.1 程序设计的基本元素

学会用 Scheme 表达"基本的计算":

- **三种表达力**:基本表达式、组合手段、抽象手段
- **代换模型**:用实参替换形参,然后求值过程体 — 直观的心智模型
- **应用序 vs 正则序**:先算参数 vs 先代入再用时算
- **特殊形式**:`if / cond / and / or / define / lambda` 不走应用序
- **陈述性 vs 过程性**:数学说"是什么",程序必须说"怎么算"
- **黑箱抽象**:用的人只看接口,不关心实现
- **块结构 + 词法作用域**:把辅助过程藏进主过程,内部 `define` + 自由变量自动找外层

**关键代码模式**:用牛顿法(海伦法)算平方根 — 通过反复"猜→改进→判断"逼近答案。**这一节后面所有数值方法的思想原型**。

**关键陷阱**:Exercise 1.6 的 `new-if` — `if` 必须是特殊形式,否则递归参数永远算不完,栈爆。

---

### 1.2 过程与它们所产生的计算过程

学会**辨认 procedure 产生的 process 形状**:

| 形状 | 时间 | 空间 | 例子 |
|---|---|---|---|
| 线性迭代 | Θ(n) | Θ(1) | fact-iter |
| 线性递归 | Θ(n) | Θ(n) | factorial |
| 树形递归 | Θ(φⁿ) | Θ(树深度) | fib(naive) |
| 对数级 | Θ(log n) | Θ(log n) | fast-expt, GCD |

**关键区分**:
- **procedure** = 静态代码(写法)
- **process** = 动态形状(行为)
- 同一个 procedure 可以产生**完全不同的形状**

**关键概念**:
- **递归过程 vs 迭代过程**:看从中间任意一步抓快照,几个变量是否能完整描述当前状态
- **尾调用优化 (TCO)**:Scheme 强制实现 — 把尾递归"跑成"迭代,空间 Θ(1)
- **增长的阶 Θ(f(n))**:抛开常数和低阶项,刻画规模放大时代价怎么变
- **时间 vs 空间**:不一定相等,可以互换(memoization)
- **应用序 vs 正则序**:Exercise 1.20 GCD,正则序产生 Fibonacci 式表达式膨胀
- **概率算法**:Fermat 测试 + Miller-Rabin — 放弃绝对确定换可证明的"几乎正确" + 指数加速

**关键工程教训**:Exercise 1.25(Alyssa)+ Exercise 1.26(Louis)— **数学等价的写法,可以导致完全不同的复杂度**。`expmod` 必须每步取模,`square` 必须用平方而不是 `(* X X)`。

### 1.3 用高阶过程构造抽象

学会**把过程当一等公民**:

#### 一等公民的四项权利

1. **能被变量命名**(`(define f square)`)
2. **能作参数传递**(`(sum cube 1 inc 10)`)
3. **能作返回值**(`(average-damp f)`)
4. **能放进数据结构**(`(list square cube)`)

#### 关键抽象工具

| 抽象 | 数学对应 | 用途 |
|---|---|---|
| `sum` | $\sum$ | 求和的所有变种 |
| `product` | $\prod$ | 求积、阶乘、Wallis 公式 |
| `accumulate` | monoid 的 fold | 任何"二元结合 + 单位元"的归约 |
| `filtered-accumulate` | map+filter+reduce | LINQ / Stream / Spark 的雏形 |
| `fixed-point` | 不动点 | 反复迭代直到稳定 |
| `average-damp` | 算法变换 | 把震荡变成收敛 |
| `deriv` | $D$(导数算子) | 函数的数值导数函数 |
| `newton-transform` | 牛顿迭代 | 解 g(x)=0 的不动点变换 |
| `fixed-point-of-transform` | 元方法 | "找方程不动点 + 用变换让它收敛"的统一壳 |
| `repeated` / `compose` | 函数幂 / ∘ | 高阶组合 |
| `iterative-improve` | 迭代改进的元模式 | 整章数值方法的元抽象 |

#### 关键洞察

- **lambda + let**:`let` 只是 lambda 应用的语法糖 — Scheme 极简哲学的体现
- **Exercise 1.34 `(f f)`**:动态类型的代价与好处,过程是普通的值,可以传给自己
- **数学等价 ≠ 算法等价**:$y=x/y$ 和 $y=(y+x/y)/2$ 解相同,行为天壤之别 — 平均阻尼让发散变收敛
- **闭包 + GC**:一等公民过程**强迫**语言提供闭包和垃圾回收 — 这就是 Lisp 1959 年发明 GC 的原因
- **抽象的复利**:`integral` 用 `sum` 后没有 `if` 没有递归;sqrt 牛顿版只有 3 行 — **代码越来越短,概念越来越清晰**

## 全章三个最深的洞察

### 一、过程不只是"代码块",而是"值"

```
1.1: square 是一个能被调用的"东西"
1.2: square 在调用时产生一个 process(可能递归可能迭代)
1.3: square 是一个值,可以被传递、返回、放进列表
```

**这个观念决定了一门语言的"抽象天花板"**。C 把函数和值分开,所以你不可能写 `average-damp` 这种东西。Scheme 把它们统一,所以 1.3 才有可能。

### 二、抽象的层级可以无限往上

```
数 → 运算 → 模式 → 算法 → 算法变换 → 元方法
1     2      3     4         5           6
```

每一层把前一层当零件用。**没有任何机制限制"还能不能再抽象一层"** — 唯一的限制是"读者还能不能理解"。

老练的程序员**知道在哪一层停下**:抽象不足重复劳动,抽象过度无人理解。这个判断力是工程经验,不是语法。

### 三、procedure 和 process 是两件事

最容易混淆但最重要的区分:

| 静态(procedure) | 动态(process) |
|---|---|
| 写在源码里的函数定义 | 运行起来真实的演化形状 |
| 棋谱 | 一局棋的实际进程 |
| 你写的 `factorial` | 它跑起来产生的 deferred 操作链 |

**写出"形状好"的过程是一种主动设计能力** — 不是随便写一个递归就完事。1.2 全章都在训练这种能力。

## 跨章一致的设计原则

整章贯穿的几条 SICP 设计哲学:

### 1. 最少机制原则

> 用最少的核心概念,通过组合得到一切。

例子:
- `let` 不是新机制 — 只是 lambda 的语法糖
- `(define (f x) ...)` 不是新机制 — 只是 `(define f (lambda (x) ...))` 的语法糖
- 控制结构(if/cond)和表达式没有本质区别 — 都是组合式

**对比**:Java 有 class、interface、abstract class、enum、record、annotation… 每个新需求加一个新关键字。Scheme 用 lambda 一种东西就能造出几乎所有东西。

### 2. 抽象的复利效应

每写一个抽象,后面就少写一类样板代码:

```
sum 写好了 → integral 一行
fixed-point 写好了 → sqrt / cube-root / 解任意方程都是配置
cont-frac 写好了 → e / tan / π / 黄金比例都是配置
average-damp 写好了 → 任何不动点搜索都能加速
```

**抽象的回报是几何级的**。

### 3. 数学和代码的对应

SICP 反复展示:**好的抽象让代码长得像数学**:

- $\Sigma$ 对应 `sum`
- $\prod$ 对应 `product`
- $D$ 对应 `deriv`
- $f \circ g$ 对应 `compose`
- $f^n$ 对应 `repeated`
- 不动点对应 `fixed-point`

这不是巧合 — 数学家几个世纪做的事(给"算子"命名),Scheme 让你直接在代码里复刻。

### 4. 严格内核 + 宽容外壳

```
half-interval-method  ← 对外:友好,自动判断符号
       │
     search           ← 对内:核心算法,接口严格
```

把易错的内核包起来,提供宽容的对外接口。这是软件工程里非常常见的模式。

## 隐性主线:为什么 SICP 第一章是经典

SICP 第一章在计算机教育史上有特殊地位。原因:

### 1. 它在第一章就把"抽象"演练到了极致

大部分编程教材第一章教 `printf`、`if`、`while`。SICP 第一章教**怎么思考抽象**。

很多大学生学完 SICP 第一章后会感到"对编程的理解被重置了" — 之前以为编程是"按步骤写指令",学完发现编程是"在不同抽象层级上设计概念"。

### 2. 它用最简单的领域(数值计算)讲最深的思想

通过"求平方根"这一个看似简单的问题,从第一节走到最后一节:

- 1.1.7 朴素的 sqrt-iter(12 行,4 个相互依赖的过程)
- 1.3.3 sqrt 作为不动点 + 平均阻尼(4 行)
- 1.3.4 sqrt 作为牛顿法(3 行)
- 1.3.4 末尾 sqrt 作为 `fixed-point-of-transform` 的特例(3 行,但和 cube-root、解任意方程都共享同一个壳)

**同一个问题被用不同抽象重做四次,每一次代码都更短、概念都更清晰**。这种"同一问题反复抽象提升"的演练,在别的教材里很少见。

### 3. 它解释了 Lisp 为什么是 Lisp

读完第一章你能理解:
- 为什么 Lisp 一定要有 GC(McCarthy 1959)— 一等公民过程强迫的
- 为什么 Lisp 是第一个有 lambda 的语言 — 抽象天花板需要它
- 为什么 Lisp 几十年慢得令人发指,然后被工业界广泛接受 — 硬件追上了设计

**Lisp 1958 年的设计选择,工业界用了 60 年才追上**。今天写 `arr.map(x => x * 2)` 的每一行 JavaScript,本质上都是 1958 年 McCarthy 那个赌注的回报。

## 第一章 → 后续章节的桥梁

SICP 后面 4 章都在用第一章建立的工具继续抽象,但抽象的对象不同:

| 章 | 抽象的对象 | 新概念 |
|---|---|---|
| **第 1 章** | 过程 (procedure) | 高阶过程、过程作值 |
| **第 2 章** | 数据 (data) | pair、列表、序列、闭包数据结构 |
| **第 3 章** | 状态 (state) | 赋值、对象、可变性、流 |
| **第 4 章** | 语言 (language) | 解释器、元循环求值器 |
| **第 5 章** | 机器 (machine) | 编译、寄存器机、显式控制 |

**核心反射"识别 → 命名 → 推广"贯穿始终**,只是"被命名的东西"不断变换:从过程,到数据,到状态,到语言,到机器。

第 2 章一开头会让你看到:**第 1 章的所有数值算法,从某种意义上只是冰山一角** — 真正的复杂性来自处理结构化数据(列表、树、图)。但 1.3 学到的高阶抽象工具(map / filter / reduce)会**直接搬到序列上**,变成第 2 章的主要武器。

## 一句话总结

> **第 1 章用最简单的数值计算领域,把"抽象"这个程序设计的核心动作演练到了极致 — 从给运算命名,到塑造过程的形状,到给"过程的模式"命名,再到给"如何制造方法的方法"命名。**
> **学完第 1 章,你不只是学会了 Scheme,你换了一个看待"代码是什么"的视角。**

## 推荐回看的关键例子

如果要从第一章里挑五段最能代表全章思想的代码:

```scheme
;; 1. sqrt 的"过程分解"风格 (1.1.7)
(define (sqrt x) (sqrt-iter 1.0 x))
(define (sqrt-iter guess x)
  (if (good-enough? guess x) guess (sqrt-iter (improve guess x) x)))

;; 2. 阶乘的两种 process 形状 (1.2.1)
(define (factorial n) (if (= n 1) 1 (* n (factorial (- n 1)))))   ; 递归
(define (fact-iter p c m) (if (> c m) p (fact-iter (* c p) (+ c 1) m)))   ; 迭代

;; 3. 高阶 sum (1.3.1)
(define (sum term a next b)
  (if (> a b) 0 (+ (term a) (sum term (next a) next b))))

;; 4. fixed-point (1.3.3)
(define (fixed-point f guess)
  (let ((next (f guess)))
    (if (close-enough? guess next) next (fixed-point f next))))

;; 5. sqrt 抽象的最高峰 (1.3.4)
(define (sqrt x)
  (fixed-point-of-transform (lambda (y) (- (square y) x))
                            newton-transform
                            1.0))
```

把这五段代码放在一起看 — **它们用层层升级的抽象解决了从"能用"到"优雅"到"通用"的同一个问题**。这就是 SICP 第一章。
