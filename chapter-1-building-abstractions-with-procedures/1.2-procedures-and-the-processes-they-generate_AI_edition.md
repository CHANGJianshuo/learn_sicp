## 全章导览

1.1 学完后,你能写出语法正确的 Scheme 程序——但你**还不会"看出"程序运行起来是什么形状**。这正是 1.2 要解决的:

> 程序员的任务是规划计算过程的演化方向,通过组合过程使过程沿着我们预期的方向展开。
> ——SICP

**核心区分**:

| | 静态 | 动态 |
|---|---|---|
| 名字 | **procedure (过程)** | **process (计算过程)** |
| 是什么 | 写在源码里的函数定义 | 程序运行起来真实的演化形状 |
| 类比 | 棋谱 | 一局棋的实际进程 |

同一个 procedure 可以产生**完全不同形状**的 process,带来截然不同的时间/空间代价。1.2 教你:

1. **辨认形状**:线性递归、线性迭代、树形递归、对数级递归
2. **量化代价**:订单(orders of growth)Θ(n)、Θ(log n)、Θ(2ⁿ) 等
3. **改造形状**:把"贵"的递归改写成"便宜"的迭代、把 Θ(n) 改成 Θ(log n)

## 1.2.1 线性递归与迭代 (Linear Recursion and Iteration)

### 阶乘的两种实现

**版本 A:线性递归**

```scheme
(define (factorial n)
  (if (= n 1)
      1
      (* n (factorial (- n 1)))))
```

代换模型展开 `(factorial 6)`:

```
(factorial 6)
(* 6 (factorial 5))
(* 6 (* 5 (factorial 4)))
(* 6 (* 5 (* 4 (factorial 3))))
(* 6 (* 5 (* 4 (* 3 (factorial 2)))))
(* 6 (* 5 (* 4 (* 3 (* 2 (factorial 1))))))
(* 6 (* 5 (* 4 (* 3 (* 2 1)))))
(* 6 (* 5 (* 4 (* 3 2))))
(* 6 (* 5 (* 4 6)))
(* 6 (* 5 24))
(* 6 120)
720
```

形状:**先膨胀,再收缩**。中间一行有 6 个待执行的 `*`,称为 **deferred operations (延迟运算)**。解释器必须记住"等内层递归回来后还要乘什么",这些信息存在调用栈里。

- 时间 Θ(n)
- 空间 Θ(n)(栈随 n 线性增长)

这就是**线性递归过程 (linear recursive process)**。

**版本 B:线性迭代**

```scheme
(define (factorial n) (fact-iter 1 1 n))

(define (fact-iter product counter max-count)
  (if (> counter max-count)
      product
      (fact-iter (* counter product)
                 (+ counter 1)
                 max-count)))
```

展开 `(factorial 6)`:

```
(fact-iter 1   1 6)
(fact-iter 1   2 6)
(fact-iter 2   3 6)
(fact-iter 6   4 6)
(fact-iter 24  5 6)
(fact-iter 120 6 6)
(fact-iter 720 7 6)
720
```

形状:**始终一条直线**。每一行只需要 `(product, counter, max-count)` 三个数,**不依赖之前的任何信息**。

- 时间 Θ(n)
- 空间 Θ(1)

这是**线性迭代过程 (linear iterative process)**。

### 核心区分:递归过程 vs 迭代过程

| 维度 | 递归过程 | 迭代过程 |
|---|---|---|
| 状态 | 隐含在调用栈里(deferred operations) | **完全由几个状态变量描述** |
| 中断恢复 | 必须连同栈一起恢复 | 给出几个状态变量就能继续 |
| 空间 | Θ(n) | Θ(1) |

**判断标准**:看从中间任意一步抓个快照,**几个变量是否能完整描述当前状态**。能 → 迭代;不能(还得知道"返回去要做什么")→ 递归。

### "递归过程" ≠ "递归函数"

这是 SICP 最容易混的一个点:

| 术语 | 含义 |
|---|---|
| **recursive procedure** | 语法上自己调用自己的函数 (syntactic property) |
| **recursive process** | 运行起来产生延迟操作的 process (behavioral property) |

**fact-iter 在语法上是递归的(自己调用自己),但产生的是迭代过程**。这一点在大多数语言里不成立,但在 Scheme 里成立——靠的是**尾调用优化**。

### Scheme 的尾调用优化 (Tail Call Optimization)

#### 普通调用 vs 尾调用

```scheme
(* counter (factorial (- counter 1)))   ; 普通调用:递归回来还要乘
(fact-iter (* counter product) ...)     ; 尾调用:递归是函数体的最后动作
```

**尾调用 (tail call)** = 函数体的"最后一步"是另一个函数调用,调用结果直接当成本函数的返回值,**不再有任何后续动作**。

#### 普通编译器怎么做

每次函数调用压一个**栈帧**,记录:返回地址、局部变量、参数。函数返回后弹栈。

`(fact-iter 1 1 6)` 调用 `(fact-iter 1 2 6)`,即使后者返回后什么都不做,也要压一帧——递归 1 万次就 1 万帧,栈溢出。

#### TCO 的把戏

Scheme 标准**强制要求**:看到尾调用,**直接复用当前栈帧**——把参数覆盖掉,跳到函数开头,不压新帧。

效果:1 亿次"递归"调用占常数空间,跟 `for` 循环一样。

#### Python 的对比

Python **没有 TCO**(吉多明确反对)。同样写法在 Python 里会栈溢出:

```python
def fact_iter(product, counter, max_count):
    if counter > max_count: return product
    return fact_iter(counter * product, counter + 1, max_count)

fact_iter(1, 1, 10000)   # RecursionError: maximum recursion depth exceeded
```

所以 Python 程序员习惯用 `while`,Scheme 程序员习惯用尾递归,**两者表达力等价**。Scheme 把"循环"和"函数调用"统一了,代码更简洁。

#### 什么是尾位置 (tail position)

```scheme
(define (f x)
  (if (< x 0)
      (g x)              ; ← 尾调用
      (+ 1 (h x))))      ; ← (h x) 不是尾调用 (返回后还要 +1)
```

判定规则:函数体执行流的"最后一个动作"是个函数调用。`if`/`cond` 各分支都可能是尾位置。

### 关系总览

```
                          ┌──── 写法 (静态) ───┐
                          │                    │
                       递归函数            非递归函数
                          │
        ┌─────────────────┴─────────────────┐
   产生递归过程                       产生迭代过程
   (有延迟操作,空间 Θ(n))           (尾递归,空间 Θ(1))
        │                                  │
        └──────────────┬───────────────────┘
                       │
                  执行行为 (动态)
```

- **递归 ↔ 迭代** = 行为对立
- **尾递归 = 一种写法**,在 TCO 语言里产生迭代过程,在非 TCO 语言里产生递归过程

### Exercise 1.9 — 两个 `+` 过程

```scheme
; 版本 A
(define (+ a b)
  (if (= a 0) b (inc (+ (dec a) b))))

; 版本 B
(define (+ a b)
  (if (= a 0) b (+ (dec a) (inc b))))
```

**版本 A**,展开 `(+ 4 5)`:

```
(+ 4 5)
(inc (+ 3 5))
(inc (inc (+ 2 5)))
(inc (inc (inc (+ 1 5))))
(inc (inc (inc (inc (+ 0 5)))))
(inc (inc (inc (inc 5))))
(inc (inc (inc 6)))
(inc (inc 7))
(inc 8)
9
```

**先膨胀,再收缩** —— 递归过程。`inc` 是延迟操作。

**版本 B**,展开 `(+ 4 5)`:

```
(+ 4 5)
(+ 3 6)
(+ 2 7)
(+ 1 8)
(+ 0 9)
9
```

**始终一条直线** —— 迭代过程。状态完全由 `(a, b)` 描述。

**关键点**:两个写法都是"递归函数",但形状完全不同。版本 B 的 `+` 在尾位置,A 的 `+` 在 `inc` 的内部不在尾位置。

### Exercise 1.10 — Ackermann 函数

```scheme
(define (A x y)
  (cond ((= y 0) 0)
        ((= x 0) (* 2 y))
        ((= y 1) 2)
        (else (A (- x 1) (A x (- y 1))))))
```

**计算几个值**:

- `(A 1 10)` = 1024
- `(A 2 4)` = 65536
- `(A 3 3)` = 65536

**几个简单数学表达**:

```scheme
(define (f n) (A 0 n))   ; → f(n) = 2n
(define (g n) (A 1 n))   ; → g(n) = 2ⁿ
(define (h n) (A 2 n))   ; → h(n) = 2 ↑↑ n  (tetration,2 的 n 次叠幂)
```

#### 为什么 g(n) = 2ⁿ

由定义,当 x=1 时:`(A 1 n) = (A 0 (A 1 (n-1))) = 2 · (A 1 (n-1))`。

加上 `(A 1 1) = 2`,得 `g(n) = 2 · g(n-1)`,即 g(n) = 2ⁿ。

#### 为什么 h(n) 是 tetration

类似:`(A 2 n) = (A 1 (A 2 (n-1))) = 2^(A 2 (n-1)) = 2^h(n-1)`。

加上 `h(1) = 2`,得:

$$h(n) = \underbrace{2^{2^{\cdots^{2}}}}_{n \text{ 个 2}}$$

具体:h(1)=2, h(2)=4, h(3)=16, h(4)=65536, **h(5) ≈ 2^65536**(远超宇宙原子数)。

这就是 Ackermann 的著名特性——**它增长得比任何"原始递归"函数都快**,常被用于证明可计算性理论里的分层结果。

| 概念 | 一句话 |
|---|---|
| procedure | 写好的代码 (静态) |
| process | 跑起来的形状 (动态) |
| 递归过程 | 有 deferred 操作,空间 Θ(n) |
| 迭代过程 | 状态变量完整,空间 Θ(1) |
| 尾调用 | 函数体最后一步就是函数调用 |
| TCO | Scheme 强制规定,把尾递归优化成迭代 |

## 1.2.2 树形递归 (Tree Recursion)

### Fibonacci 的朴素递归

```scheme
(define (fib n)
  (cond ((= n 0) 0)
        ((= n 1) 1)
        (else (+ (fib (- n 1))
                 (fib (- n 2))))))
```

`(fib 5)` 的调用树:

```
                    fib 5
                  /       \
               fib 4      fib 3
              /    \      /    \
          fib 3  fib 2  fib 2  fib 1
         /  \    / \    / \
       ...  ... ... ...
```

**每个调用产生两个子调用**——这是**树形递归 (tree recursive)**。

### 复杂度

- **时间 Θ(φⁿ)**,φ = (1+√5)/2 ≈ 1.618(黄金比例)
- **空间 Θ(n)**(注意!不是 Θ(φⁿ))

#### 为什么时间是 φⁿ

设 T(n) 为 fib 调用次数,有 T(n) = T(n-1) + T(n-2) + 1,**结构与 Fibonacci 本身相同**,所以增长速率也是 φⁿ。

#### 为什么空间只有 Θ(n)

> **关键**:Scheme 是深度优先遍历,任何时刻**只有从根到叶的一条路径**在栈里。

虽然整个树有 φⁿ 个节点,但树的**深度**只有 n 左右。栈的最大深度 = 树的最大深度 = Θ(n)。

类比:你在迷宫里探索一个巨大的二叉分叉路径。即使总路径数是天文数字,你身上同时只背着"当前从入口走过来的那一条路径上的标记"——下次回到岔路口要找的标记数 = 路径深度,不是节点总数。

时间和空间不对称是树形递归的标志特征:
- **时间** = 计算的总工作量 = 节点数
- **空间** = 同时挂起的状态量 = 路径深度

### Fibonacci 的迭代版

```scheme
(define (fib n) (fib-iter 1 0 n))

(define (fib-iter a b count)
  (if (= count 0)
      b
      (fib-iter (+ a b) a (- count 1))))
```

不变量:`a = Fib(k+1)`, `b = Fib(k)`,每步往前推一格。

- 时间 Θ(n)
- 空间 Θ(1)

**树形递归到线性迭代:从 Θ(φⁿ) 到 Θ(n),指数级压缩**。

### 三种形状的递进

| 形状 | 时间 | 空间 | 例子 |
|---|---|---|---|
| 线性迭代 | Θ(n) | Θ(1) | fact-iter |
| 线性递归 | Θ(n) | Θ(n) | factorial |
| 树形递归 | Θ(φⁿ) (或更多) | Θ(树深度) | fib |

**树形和线性的本质区别**:
- 线性 = 每步**一个**子问题
- 树形 = 每步**多个**子问题(分支)

#### 为什么不会有"四叉""五叉"形状?

会有,但通常都被归到"树形递归"这一大类。重点不是分几叉,而是"每步是 1 个还是多个子问题"。如果树有重复子问题(像 fib),就有**指数级冗余计算**——这正是引出**记忆化 / DP** 的动机(下一节会用 counting change 演示)。

---

### 例子:换零钱 (Counting Change)

**问题**:用 50¢、25¢、10¢、5¢、1¢ 五种硬币给 1 美元找零,有多少种方法?

#### 巧妙的递归分解

把"用 n 种硬币凑 a 元的方法数"分成两堆,**没有重叠没有遗漏**:

> 方法数 = 不用第一种硬币的方法数 + 至少用一次第一种硬币的方法数
>       = (用其余 n-1 种凑 a)
>       + (用所有 n 种凑 a-d)   d 是第一种硬币的面值

数学语言:`cc(a, n) = cc(a, n-1) + cc(a-d, n)`。

边界:
- `a = 0` → 1(凑零有"什么都不用"这一种)
- `a < 0` 或 `n = 0` → 0(凑不出来)

```scheme
(define (count-change amount) (cc amount 5))

(define (cc amount kinds-of-coins)
  (cond ((= amount 0) 1)
        ((or (< amount 0) (= kinds-of-coins 0)) 0)
        (else (+ (cc amount (- kinds-of-coins 1))
                 (cc (- amount (first-denomination kinds-of-coins))
                     kinds-of-coins)))))

(define (first-denomination kinds-of-coins)
  (cond ((= kinds-of-coins 1) 1)
        ((= kinds-of-coins 2) 5)
        ((= kinds-of-coins 3) 10)
        ((= kinds-of-coins 4) 25)
        ((= kinds-of-coins 5) 50)))

(count-change 100)   ; → 292
```

#### 风格点

- 这是**树形递归**,大量重复子问题
- 在小输入下能跑,但 `(count-change 1000)` 会非常慢
- **优化方向**:记忆化(把已算的 `cc(a, n)` 存表),把指数变多项式
- 这正是动态规划 (DP) 的入口

### Exercise 1.11

函数定义:

$$f(n) = \begin{cases} n & n < 3 \\ f(n-1) + 2f(n-2) + 3f(n-3) & n \geq 3 \end{cases}$$

**递归版**:

```scheme
(define (f n)
  (if (< n 3)
      n
      (+ (f (- n 1))
         (* 2 (f (- n 2)))
         (* 3 (f (- n 3))))))
```

**迭代版**:用三个滚动变量保存最近三个值。

```scheme
(define (f n) (f-iter 2 1 0 n))

(define (f-iter a b c count)
  (if (= count 0)
      c
      (f-iter (+ a (* 2 b) (* 3 c))
              a
              b
              (- count 1))))
```

不变量:`(a, b, c) = (f(k+2), f(k+1), f(k))`,每步往前推一格。

时间从 Θ(αⁿ)(α 是某个三阶递归的特征根) 降到 Θ(n)。

### Exercise 1.12 — Pascal 三角

```
    1
   1 1
  1 2 1
 1 3 3 1
1 4 6 4 1
```

递归定义:`P(r, c) = P(r-1, c-1) + P(r-1, c)`,边界 `P(r, 0) = P(r, r) = 1`。

```scheme
(define (pascal row col)
  (cond ((or (= col 0) (= col row)) 1)
        (else (+ (pascal (- row 1) (- col 1))
                 (pascal (- row 1) col)))))
```

闭式:$P(r, c) = \binom{r}{c}$。

这又是个**树形递归**,有重叠子问题——是经典 DP 入门题。

### Exercise 1.13 — Fibonacci 闭式

证明:$\text{Fib}(n)$ 是 $\frac{\varphi^n}{\sqrt{5}}$ 离最近整数,其中 $\varphi = \frac{1+\sqrt{5}}{2}$。

#### Binet 公式

$$\text{Fib}(n) = \frac{\varphi^n - \psi^n}{\sqrt{5}}, \quad \psi = \frac{1-\sqrt{5}}{2} \approx -0.618$$

#### 关键观察

- $\varphi$ 和 $\psi$ 都是 $x^2 = x + 1$ 的根
- $|\psi| < 1$,所以 $\psi^n$ 随 n 增长**指数衰减**到 0
- 因此 $\text{Fib}(n) \approx \frac{\varphi^n}{\sqrt{5}}$,误差 $\frac{|\psi|^n}{\sqrt{5}} < \frac{1}{2}$,所以四舍五入恰好得整数

#### 归纳证明骨架

基础:$n=0,1$ 直接验证。

归纳:假设公式对 n、n-1 成立,要证对 n+1 成立。

$$\text{Fib}(n+1) = \text{Fib}(n) + \text{Fib}(n-1) = \frac{\varphi^n - \psi^n + \varphi^{n-1} - \psi^{n-1}}{\sqrt 5}$$

利用 $\varphi^2 = \varphi + 1 \Rightarrow \varphi^{n+1} = \varphi^n + \varphi^{n-1}$,同理对 $\psi$:

$$= \frac{\varphi^{n+1} - \psi^{n+1}}{\sqrt 5} \quad \blacksquare$$

#### 启示

Fibonacci 的指数增长率正好是 $\varphi$。这就是为什么朴素 fib 是 Θ(φⁿ) 时间。

## 1.2.3 增长的阶 (Orders of Growth)

### Θ(f(n)) 的定义

> 我们说 R(n) 的增长阶是 Θ(f(n)),记 R(n) = Θ(f(n)),如果存在常数 k₁、k₂ 使得对所有足够大的 n:
> $$k_1 f(n) \leq R(n) \leq k_2 f(n)$$

直觉:**抛开常数倍数和低阶项,看 R(n) 跟 f(n) 同一个量级**。

### 通俗版

设想一个工厂,n 是订单数,R(n) 是处理 n 个订单要花的小时数。Θ(f(n)) 不告诉你具体几小时,只告诉你"**当订单翻倍时,工时按什么规律变化**":

| 阶 | 工时变化 | 类比 |
|---|---|---|
| Θ(1) | 不变 | 无论几个订单都是 1 小时 |
| Θ(log n) | 加一点点(订单翻倍只多 1 小时) | 二分查字典 |
| Θ(n) | 翻倍(线性) | 一个一个数土豆 |
| Θ(n log n) | 略多于翻倍 | 排序 |
| Θ(n²) | 翻 4 倍(平方) | 比对所有 pair |
| Θ(2ⁿ) | 翻平方倍(指数) | 穷举所有子集 |

**关键**:增长阶决定了"问题规模能扩大多少",不是"小规模下哪个快"。常数项可能让 Θ(n²) 在 n=10 时比 Θ(n log n) 快,但 n=10000 时一定输。

### 时间 vs 空间

- **时间复杂度** = 总操作次数 ≈ 跑多久
- **空间复杂度** = 同时占用的最大内存 ≈ 多大内存

时间和空间**不一定相等**:
- 线性递归 fact:时间 Θ(n),空间 Θ(n)
- 线性迭代 fact-iter:时间 Θ(n),空间 Θ(1)
- 树形递归 fib:时间 Θ(φⁿ),空间 Θ(n)

**用空间换时间**(memoization、缓存)、**用时间换空间**(重新计算而不存储)是经典权衡。

### 这一节的作用

1.2.3 是后面所有讨论的**度量尺**。从这里起,描述算法不说"很快""有点慢",改说 Θ(log n) / Θ(n) / Θ(n²) ——一种**精确而抽象**的语言。

## 1.2.4 求幂 (Exponentiation)

### 三个版本

**线性递归**:

```scheme
(define (expt b n)
  (if (= n 0)
      1
      (* b (expt b (- n 1)))))
```

时间 Θ(n),空间 Θ(n)。

**线性迭代**:

```scheme
(define (expt b n) (expt-iter b n 1))

(define (expt-iter b counter product)
  (if (= counter 0)
      product
      (expt-iter b (- counter 1) (* b product))))
```

时间 Θ(n),空间 Θ(1)。

**对数级:连续平方法**

```scheme
(define (fast-expt b n)
  (cond ((= n 0) 1)
        ((even? n) (square (fast-expt b (/ n 2))))
        (else (* b (fast-expt b (- n 1))))))
```

时间 **Θ(log n)**,空间 Θ(log n)(递归过程)。

### 为什么是对数

每次 `n` 偶数时除以 2,奇数时减 1 后变成偶数。所以每两步 n 至少减半,总步数 = Θ(log n)。

例:b¹⁰⁰⁰ 只要约 10 次乘法(因为 1000 ≈ 2¹⁰),不是 1000 次。

### 这是密码学的基础

模幂运算 `bⁿ mod m` 用同样思路就是 RSA/ECC/Diffie-Hellman 的核心算子。1.2.6 会用它实现 Fermat 测试。

### Exercise 1.20 — 正则序 vs 应用序的 GCD

GCD 的迭代实现:

```scheme
(define (gcd a b)
  (if (= b 0)
      a
      (gcd b (remainder a b))))
```

#### 应用序(Scheme 默认)

每步先把 `(remainder a b)` 算成数字再代入:

```
(gcd 206 40)
→ (gcd 40 (remainder 206 40))     算一次 → 6     【1 次】
→ (gcd 40 6)
→ (gcd 6 (remainder 40 6))        算一次 → 4     【2 次】
→ (gcd 6 4)
→ (gcd 4 (remainder 6 4))         算一次 → 2     【3 次】
→ (gcd 4 2)
→ (gcd 2 (remainder 4 2))         算一次 → 0     【4 次】
→ (gcd 2 0)
→ 2
```

**应用序:4 次 remainder。**

#### 正则序(代换、用时再算)

参数表达式**原样代入**,只有需要值(`if` 的条件)时才算。

记 `r(x,y) = (remainder x y)` 简写。每步要判断 `b = 0`,触发 `b` 表达式求值。

| 步骤 | b 表达式 | 求值 b 的 remainder 次数 |
|---|---|---|
| 0 | `40` | 0 |
| 1 | `r(206,40)` | 1 |
| 2 | `r(40, r(206,40))` | 2 |
| 3 | `r( r(206,40),  r(40, r(206,40)) )` | 4 |
| 4 | (更深嵌套,共 7 次) | 7 |
| 返回 a | a 是上一步的复杂表达式,需求值 | 4 |

**总计:18 次 remainder。**

#### 为什么差距这么大

设第 k 步 `b` 表达式中含 $f_k$ 个 `remainder`,递推:

$$f_k = f_{k-1} + f_{k-2} + 1$$

**Fibonacci 式增长** —— 每多走一步,表达式复杂度近似乘以 φ。

应用序之所以高效,是因为**中间值算出来一次就被复用**;正则序里同一个 `b` 表达式被**展开多次**,重复计算。

#### "正则序"的来历

英文 **normal order**。"正则"是日语对 normal 的翻译,经过中文计算机术语沿袭。"normal" 来自 lambda 演算的 **normal form**(正规形式)——一个 lambda 项不断 β-归约,所到达的不可再化简形式。

**Normal-order reduction** = 总是先归约最外层的 redex,这是定理保证"如果存在 normal form 就一定能找到"的归约策略。SICP 里的"代换 + 不到必要不算"正是这个策略在求值层面的体现。

### 现实修补

纯正则序代价太高。Haskell 用 **call-by-need**:正则序 + **memoization**(第一次算完结果存住,后续直接取)。这样既保留了"不需要的值不算"的好处,又避免了重复计算。

## 1.2.5 最大公约数 (Greatest Common Divisors)

```scheme
(define (gcd a b)
  (if (= b 0)
      a
      (gcd b (remainder a b))))
```

**欧几里得算法**:`gcd(a, b) = gcd(b, a mod b)`,用辗转相除直到余数为 0。

- 时间 **Θ(log n)**(Lamé 定理:步数与 b 的位数成正比;最坏情况是相邻 Fibonacci 数)
- 空间 Θ(1)(尾递归)

**Lamé 定理**:若欧几里得算法对 (a, b) 需要 k 步,则 b ≥ Fib(k)。

由于 Fib(k) ≈ φᵏ/√5,所以 k ≤ log_φ(√5 · b) = Θ(log b)。

这是 SICP 里第一次出现"算法步数和 Fibonacci 数挂钩"的真实定理,展示了数论与算法分析的深刻联系。

## 1.2.6 例子:素性测试 (Testing for Primality)

两种方法:

| 方法 | 复杂度 | 性质 |
|---|---|---|
| 试除法 | Θ(√n) | 确定性 |
| Fermat 测试 | Θ(log n) | 概率性 |

### 试除法

```scheme
(define (smallest-divisor n) (find-divisor n 2))

(define (find-divisor n test-divisor)
  (cond ((> (square test-divisor) n) n)
        ((divides? test-divisor n) test-divisor)
        (else (find-divisor n (+ test-divisor 1)))))

(define (divides? a b) (= (remainder b a) 0))

(define (prime? n) (= n (smallest-divisor n)))
```

#### 为什么只查到 √n 就够

若 n 合数,n = a · b,a 和 b 不可能都 > √n(否则乘积 > n)。所以**至少有一个因子 ≤ √n**。

#### 用 `(square test-divisor) > n` 而不是 `test-divisor > (sqrt n)`

1. **避免浮点精度问题**:`sqrt` 返回浮点
2. **避免重复计算**:每次循环都算一次 sqrt 是浪费

#### 复杂度的真相

Θ(√n) 看着不算大,但相对**输入位数 d = log n** 是 **指数级**:

$$\sqrt n = 10^{d/2}$$

100 位的数(RSA 级别)要 10⁵⁰ 次试除,**宇宙寿命算不完**。这就需要下面的对数算法。

### Fermat 测试

#### Fermat 小定理

> n 素数 ⇒ 对任意 a < n,$a^n \equiv a \pmod n$

反过来用:随机选 a,算 $a^n \bmod n$。
- ≠ a → **n 一定不是素数**
- = a → **n 大概率是素数**,多试几次提高可信度

#### `expmod` 模幂运算

```scheme
(define (expmod base exp m)
  (cond ((= exp 0) 1)
        ((even? exp)
         (remainder (square (expmod base (/ exp 2) m)) m))
        (else
         (remainder (* base (expmod base (- exp 1) m)) m))))
```

结构同 `fast-expt`,**每步都模一下**,中间数永远 < m²,Θ(log n)。

利用数论性质:$(a \cdot b) \bmod m = ((a \bmod m) \cdot (b \bmod m)) \bmod m$。

#### Fermat 测试本身

```scheme
(define (fermat-test n)
  (define (try-it a)
    (= (expmod a n n) a))
  (try-it (+ 1 (random (- n 1)))))

(define (fast-prime? n times)
  (cond ((= times 0) true)
        ((fermat-test n) (fast-prime? n (- times 1)))
        (else false)))
```

时间 **Θ(log n)** 单次。多次测试把误判概率推到任意小。

### 概率算法

#### Carmichael 数:Fermat 测试的死穴

存在合数 n,对**所有** a < n 都满足 $a^n \equiv a \pmod n$。例:561, 1105, 1729, 2465。

对它们,Fermat 测试 100% 误判,无论测多少次。

#### Miller-Rabin:不能被骗

练习 1.28 实现的算法。利用"1 模 n 的非平凡平方根":若发现 $x^2 \equiv 1 \pmod n$ 但 $x \neq \pm 1$,则 n 一定不是素数。

定理保证:**任何合数 n,至少一半的 a 会触发非平凡平方根或 $a^{n-1} \not\equiv 1$**。

所以 k 次测试的误判概率 < (1/2)ᵏ,可证可控。

#### 范式意义

> **放弃绝对确定,换取可证明的"几乎正确" + 指数级加速。**

类别:
- **Monte Carlo**:答案可能错,时间确定 (Miller-Rabin)
- **Las Vegas**:答案永远对,时间随机 (随机化快排)

应用:密码学(RSA 素数生成)、Bloom filter、跳表、Karger 最小割、MCMC、模拟退火。

### Exercise 1.23 — 跳过偶数,只省一半

```scheme
(define (next n)
  (if (= n 2) 3 (+ n 2)))
```

把 `(+ test-divisor 1)` 改成 `(next test-divisor)`。

**理论预期 2 倍速**,**实测常只 1.4–1.7 倍**。

**原因**:每步多了一次函数调用 + 比较,**单步常数代价变大**。

$$\text{加速比} = \frac{2c}{c'} < 2 \quad (c' > c)$$

**教训**:渐近复杂度只看步数,不看每步常数。算法层优化能否兑现取决于常数项。

### Exercise 1.25 — Alyssa 的"简化"

```scheme
(define (expmod base exp m)
  (remainder (fast-expt base exp) m))
```

数学等价,**实际不可用**。

`(fast-expt base exp)` 算出**完整的** $a^n$,对 $n = 10^{20}$ 这是 $10^{20}$ 位的数:
1. **存不下**:宇宙没那么多原子
2. **乘法不再是常数时间**:大整数乘法代价随位数增长

原版**每次乘完立刻取模**,把中间数永远卡在 < m²,所以每步真常数,整体 Θ(log n)。

### Exercise 1.26 — Louis 的"展开"

把 `(square (expmod ...))` 改成 `(* (expmod ...) (expmod ...))`。

**应用序会算两次** expmod,递归树从一条直线变成完整二叉树。

| 版本 | 递推 | 复杂度 |
|---|---|---|
| 原版 | $T(n) = T(n/2) + O(1)$ | Θ(log n) |
| Louis 版 | $T(n) = 2T(n/2) + O(1)$ | Θ(n) |

按主定理(Master Theorem)直接出结果。指数加速被退化回线性。

**陷阱本质**:`square` 把参数算一次再自乘;`(* X X)` 把 X 表达式算两次。这正是 Exercise 1.20 正则序问题的同构变体。

(define (full-fermat-test n)

### Exercise 1.28 — Miller-Rabin

修改 expmod,在 even 分支额外检查"1 的非平凡平方根",发现就返回 0(信号):

```scheme
(define (expmod base exp m)
  (cond ((= exp 0) 1)
        ((even? exp)
         (let ((x (expmod base (/ exp 2) m)))
           (let ((sq (remainder (square x) m)))
             (if (and (= sq 1)
                      (not (= x 1))
                      (not (= x (- m 1))))
                 0
                 sq))))
        (else
         (remainder (* base (expmod base (- exp 1) m)) m))))

(define (miller-rabin-test n)
  (define (try-it a)
    (= (expmod a (- n 1) n) 1))
  (try-it (+ 2 (random (- n 2)))))
```

返回 0 是聪明的"信号":0 不可能是合法的模 n 余数,一旦某层返回 0,会被一路平方/相乘传出去($0^2=0$,$0 \cdot x = 0$),在最后 `(= ... 1)` 处自动判失败。

**关键性质**:即便对 Carmichael 数也**至少一半** a 失败,误判概率 < (1/2)ᵏ。

这是现代密码学库(OpenSSL、GMP)真正使用的算法。

## 本章核心思想总结

| 概念 | 一句话 |
|---|---|
| procedure vs process | 静态代码 vs 动态形状,同一过程可产生不同形状 |
| 递归过程 | deferred 操作链,空间 Θ(n) |
| 迭代过程 | 状态变量完整,空间 Θ(1) |
| 尾调用优化 (TCO) | Scheme 强制实现,把尾递归优化成迭代 |
| 树形递归 | 每步多个子问题,时间膨胀但空间 = 树深度 |
| 增长阶 Θ(f(n)) | 抛开常数和低阶项,刻画"规模放大时代价怎么变" |
| 时间 vs 空间 | 不一定相等,可以互换 |
| 连续平方法 | 把 Θ(n) 变 Θ(log n) 的核心技巧 |
| 模幂 | 每步取模避免大整数,密码学的核心算子 |
| 应用序 vs 正则序 | 算参数的时机,关乎中间值能否复用 |
| 概率算法 | 放弃绝对确定换可证明"几乎正确" + 指数加速 |
| Carmichael / Miller-Rabin | Fermat 测试的漏洞 → 现代算法的诞生 |

**1.2 是 SICP 教学的真正"开关时刻"**:从这里起,"写出能跑的代码"不再是目标,**理解程序运行的形状、并主动塑造它**才是。后面所有抽象、高阶函数、数据结构、解释器,都是对这一节"塑造形状"思想的不同维度延伸。
